アーティキュレーション自動設定機能 設計方針

作成日: 2026-07-22 / ステータス: 設計確定・実装着手

1. 目的

ピッチリストの KitPitch.ArticulationID （Class/MapModels/KitPitch.cs）を、 現在の手動選択に加えて、マップデータ内の情報から自動的に割り当てられるようにする。

2. 対象外フィルタ（前処理）

Plugin.PluginType が GM / GM2 / GS / XG （Class/MapModels/PluginType.cs）の場合、GM標準ドラムマップ上でパーカッション用途に 割り当てられているノート番号に該当する KitPitch は自動設定の対象から除外する。

  • Tambourine = 54
  • Cowbell = 56
  • Vibraslap = 58
  • Bongo = 60-61
  • Conga = 62-64
  • Timbale = 65-66
  • Agogo = 67-68
  • Cabasa = 69
  • Maracas = 70
  • Whistle = 71-72
  • Guiro = 73-74
  • Claves = 75
  • Wood Block = 76-77
  • Cuica = 78-79
  • Triangle = 80-81

通常のドラム音源プラグイン（GM系以外）はこの除外を適用しない。

3. パーツ名辞書（正規化）

  • Kick/Bass Drum/BD、Hi-Hat/HH等の表記ゆれを吸収する別名辞書を用意する。
  • 用途は2つ。
    1. KitPitch.Name からどの Part（楽器）を指すか特定する。
    2. 正規化後の文字列でのマッチング精度を向上させる。
  • 永続化： アプリ全体共通設定として Setting.json に保存する。 ISettingService / SettingService の既存パターンを流用し、新規ファイル・新規I/O基盤は追加しない。
  • 編集UI： 当初は専用ページをハンバーガーメニューに追加する方針だったが、 Settingページに十分な余裕があることを確認できたため、Settingページ内のグループボックス 「Part Name Dictionary」として統合した（専用ページ・専用ハンバーガーメニュー項目は削除）。 編集用のDataGrid・Add/Deleteコマンドは SettingViewModel / SettingPage.xaml に実装する。

4. マッチング優先順位

正規化後の名前を用いて、以下の順で試行し、最初にヒットした結果を採用する。

優先度方法信頼度備考
1a 同一Plugin内の他KitでName一致 & ArticulationID設定済みのものを参照 最高 Articulation.ID はMapData全体でグローバル一意なため、Part特定不要でそのままID転用可
1b 1aでヒットしなければ、他Plugin含む全Kitを対象に同様に検索 高 別製品間の同名衝突リスクはあるが、完全一致のみを対象とすることでリスクを抑制
2 辞書正規化で対象Partを特定し、そのPartの Articulations 一覧内でName完全一致を検索 中〜高 初出の名前でも辞書があれば機械的に解決可能
3 1・2でヒットしない場合のみ、対象Part内 Articulations とのトークン一致率等でベスト候補をスコアリング 低（フォールバック） 確信度が高ければ自動適用、低い・複数候補拮抗時はユーザー確認

5. 代表ピッチ／重複ピッチの後処理

同一 ArticulationID が複数の KitPitch に割り当てられた結果になった場合：

  1. 候補の中にGM標準ドラムマップの対応ノート番号と Pitch が一致するものがあれば、それを代表とする。
  2. 該当が無ければ、最初に見つかったピッチを代表とする。
  3. 代表ピッチには Separator.None、それ以外には Separator.Duplicate を設定する。

また、ピッチ名が設定されているにもかかわらずアーティキュレーションが割り当たらなかった場合 （GM/GS/XGパーカッション除外、または優先度1〜3のいずれでも候補が見つからなかった場合）は、 Separator.Separate を設定し、未設定ピッチと区別する。

6. UI操作

KitPitchesPanel.xaml の既存コンテキストメニュー内、ContextKitPitchesCopyArticulation （アーティキュレーションコピー）の近くに新規メニュー項目「アーティキュレーション自動引き当て」を追加する。 実行対象は選択状態に関わらずKit全体（全128ピッチ）とし、既存の RelayCommand パターンで実装する。

コンテキストメニューの末尾には「アーティキュレーションとセパレータをクリア」コマンドを追加する。 Kit全体（選択不問）を対象に、全ピッチの ArticulationID を0、Separator を None に戻す（ピッチ名は変更しない）。
